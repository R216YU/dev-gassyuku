## 合宿テーマ：AI自律型「動的メディア・アセット・パイプライン」
素材が「1つ」あれば、それをAIが加工・増殖させ、NFS経由でS3に書き込み、CloudFrontで即座に公開されるまでの**「流れ（Pipeline）」**を構築します。

### 1. 最小構成のワークフロー(作業イメージ)
1.  **Input:** あなたがスマホで撮った「1枚の写真」または「1つの短い動画」をS3に手動で上げる。
2.  **Processing (Bedrock):** * **分析:** Bedrock（Multimodalモデル）が素材の内容を理解。
    * **生成:** その内容に基づいた「説明文(HTML)」「多言語翻訳」「関連するメタデータ」を生成。
3.  **Storage (S3 NFS):** LambdaがBedrockの回答を受け取り、NFSマウントされたパス（`/mnt/s3/out/`）に直接ファイルを書き出す。
4.  **Delivery (CloudFront):** 書き出された瞬間、CloudFront経由で「公開済みアセット」としてブラウザから閲覧可能にする。

## 2. 「パイプライン」を評価するための3つのチェックポイント(ゴール)

合宿のゴールを以下の「技術的検証の成功」に置くと、素材の数に関わらず達成感が出ます。

### ① 「ファイル操作」として完結しているか？
S3 API（SDK）を使わず、**Pythonの `open()` や `write()` といった標準的なファイル操作**だけでS3に保存できているか。これがNFS機能（Amazon S3 Files）を使う最大の意義です。
> **メリット:** 将来的にAIが「既存の画像編集ライブラリ（Pillowなど）」や「動画編集ツール（FFmpeg）」をローカルコマンドとして叩けるようになります。

### ② CloudFront のキャッシュ制御ができているか？
AIが同じファイル名で中身を更新した際、CloudFrontが古いデータを出し続けないように「Invalidation（キャッシュクリア）」を自動で行うパイプラインになっているか。

### ③ Bedrock の出力を直接デプロイできているか？
AIの回答（テキスト）を一度人間の手で修正するのではなく、プログラムがパースしてそのままHTML/JSONファイルとしてS3の公開領域に「自動配置」できているか。
