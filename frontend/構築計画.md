# Next.js ファイルアップロード画面 実装計画

## 概要

`frontend/` 配下に Next.js + TypeScript + Material UI でファイルアップロード画面を構築します。  
環境は **local** と **dev** の2種類。local 環境では `apigateway.yml` を元にモックサーバーを立ち上げます。

---

## API 仕様（apigateway.yml より）

| 項目 | 内容 |
|---|---|
| エンドポイント | `POST /upload` |
| リクエスト形式 | `multipart/form-data` |
| フィールド | `file` (binary) / `filename` (string → 選択ファイル名を自動設定) |
| 成功レスポンス | `200` `{ "message": "ファイルをアップロードしました" }` |
| エラーレスポンス | `400` (不正なリクエスト) / `500` (サーバーエラー) |

---

## 環境構成

| 環境 | ファイル | API URL |
|---|---|---|
| local | `.env.local` | `http://localhost:4010` (Prism モックサーバー) |
| dev | `.env.development` | `https://dev.example.com` (実際のドメインに差し替え) |

### モックサーバー（Prism）

`apigateway.yml` を OpenAPI v3 として認識させ、`@stoplight/prism-cli` でモックサーバーを起動します。

```bash
# ローカルでのモックサーバー起動コマンド（npm script として登録）
npm run mock
# → prism mock ./apigateway.yml --port 4010
```

> [!NOTE]
> `apigateway.yml` の先頭が `penapi:` になっていますが、正しくは `openapi:` です。  
> ファイルを修正するか、Prism 起動時にエラーが出た場合は修正が必要です。

## ファイル制限

| 項目 | 内容 |
|---|---|
| 許可する種別 | 画像（`image/*`）・動画（`video/*`）のみ |
| サイズ制限 | なし（APIに依存） |
| `filename` | 選択したファイルの元ファイル名を自動設定 |

---

## 実装する画面・コンポーネント

### アップロード画面 (`/`)

Material UI を使ったリッチなデザインで以下を実装：

- **ドラッグ＆ドロップエリア** — ファイルをドロップ or クリックで選択（画像・動画のみ）
- **選択ファイルプレビュー** — 画像はサムネイル表示、動画はファイル名とアイコン表示
- **アップロードボタン** — API (POST /upload) を呼び出し
- **プログレス表示** — アップロード中のローディング（`LinearProgress`）
- **結果通知** — 成功/エラー時の `Snackbar` + `Alert`

## 実装ステップ

1. **[NEW]** `frontend/app/` に Next.js プロジェクトを作成  
   `create-next-app` を npm で実行（TypeScript・App Router・src ディレクトリ）

2. **[NEW]** MUI・axios・Prism をインストール  
   - `npm install @mui/material @mui/icons-material @emotion/react @emotion/styled axios`  
   - `npm install -D @stoplight/prism-cli`

3. **[NEW]** 環境別 `.env` ファイルを作成  
   - `.env.local` → `NEXT_PUBLIC_API_BASE_URL=http://localhost:4010`  
   - `.env.development` → `NEXT_PUBLIC_API_BASE_URL=https://dev.example.com`

4. **[NEW]** `package.json` に `mock` スクリプトを追加  
   `"mock": "prism mock ../apigateway.yml --port 4010"`

5. **[NEW]** `src/config/env.ts` — 環境変数アクセサを作成

6. **[NEW]** `src/lib/api.ts` — `/upload` API クライアントを作成  
   - `filename` は `file.name` を自動設定

7. **[NEW]** `src/components/FileUpload.tsx` — アップロードコンポーネントを作成  
   - ドラッグ＆ドロップ対応、画像/動画バリデーション、プレビュー

8. **[MODIFY]** `src/app/layout.tsx` — MUI ThemeProvider を設定

9. **[MODIFY]** `src/app/page.tsx` — アップロード画面を組み込み

## 開発環境要件

- Node.js バージョン: v24系の最新版を使用すること
- ローカル起動ポート: 4444（他システムとのポート衝突回避のため）

## 検証計画

### モックサーバー確認
```bash
cd frontend/app
npm run mock   # Prism モックサーバー起動
```

### 画面確認
```bash
npm run dev    # Next.js 開発サーバー起動（.env.local が読み込まれる）
```
ブラウザで `http://localhost:3000` にアクセスし以下を確認：
- ファイル選択・ドラッグ＆ドロップが動作するか
- 画像・動画以外のファイルが弾かれるか
- アップロード時に `POST http://localhost:4010/upload` が呼ばれるか
- 成功/エラーの Snackbar が表示されるか
