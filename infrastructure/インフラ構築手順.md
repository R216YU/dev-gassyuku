# AWSインフラ構築手順

本プロジェクトのインフラ構築はAWS CloudFormationを使用して行います。

## VPCネットワークの構築 (最小コスト構成)

コストを最小限に抑えつつ、VPC内のLambdaからS3へ通信可能な最小構成のVPCを構築します。

### 事前準備
1. AWS CLIがインストールされていること。
2. デプロイ先のAWSアカウントに対する認証情報（`aws configure` 等）が設定され、権限を持っていること。

### デプロイ手順

以下のコマンドを実行してCloudFormationスタックを作成します。
※コマンドはプロジェクトルートディレクトリで実行することを想定しています。

```bash
aws cloudformation create-stack \
  --stack-name dev-gassyuku-vpc \
  --template-body file://infrastructure/vpc-serverless_none.yaml \
  --parameters file://infrastructure/vpc-serverless_none-parameters.json \
  --region ap-northeast-1
```

#### デプロイ内容
- 1つのVPC
- 1つのプライベートサブネット (1AZ構成)
- VPC内から無料・安全にS3にアクセスするための S3 Gateway Endpoint
- Lambda等にアタッチするためのセキュリティグループ

#### 確認方法
AWSマネジメントコンソールからCloudFormationの画面を開き、`dev-gassyuku-vpc` スタックのステータスが `CREATE_COMPLETE` になれば構築完了です。
