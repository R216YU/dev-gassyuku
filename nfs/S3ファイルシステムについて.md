# Amazon S3 Files（S3 ファイルシステム）概要

> 参考: [Launching S3 Files: Making S3 Buckets Accessible as File Systems](https://aws.amazon.com/jp/blogs/news/launching-s3-files-making-s3-buckets-accessible-as-file-systems/)

---

## どのような機能か

**S3 Files** は、S3 バケットをネイティブのファイルシステムとしてマウントできる AWS の新機能です。  
オブジェクトストレージである S3 に対して、**NFS v4.1 以降**でファイルシステムとしてアクセスできるようになります。

| 特徴 | 内容 |
|---|---|
| プロトコル | NFSv4.1 以降 |
| 基盤 | Amazon EFS のインフラを活用 |
| レイテンシー | アクティブなデータに対して約 1 ミリ秒 |
| 整合性モデル | NFS close-to-open 整合性（同時アクセス対応） |
| 対応サービス | EC2 / ECS / EKS / Lambda |
| OS 要件 | Linux（NFS クライアント必須） |

### 動作の仕組み

- ファイル操作（読み書き）は**自動的に S3 バケットに同期**される
- 小さなファイルや書き込みは高性能ストレージ（EFS ベース）を経由
- **128KB 以上の大きなファイルの読み取り**は S3 から直接提供（低コスト）
- データを複製せずに複数のコンピューティングリソース間で共有アクセスが可能

### 主なユースケース

- ML トレーニングパイプライン（複数ノードでデータ共有）
- エージェント型 AI（ファイルベースの Python ライブラリやシェルスクリプトを利用）
- 複数リソースが同時にデータを読み書きするワークロード
- 既存のファイルシステムベースのアプリケーションをそのまま S3 に対して動作させる

## マウントターゲット

**VPC 内に配置されるネットワークエンドポイント**で、EC2 インスタンスなどが S3 ファイルシステムに接続するための接点です。

- AZ（サブネット）ごとに作成する
- **コンソール**: ファイルシステム作成時に自動作成される
- **CLI**: `create-mount-target` コマンドで作成
- **IaC（CloudFormation / Terraform）**: テンプレートで定義可能

```bash
# NFS クライアントでのマウント例
sudo mount -t nfs4 \
  -o nfsvers=4.1,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2 \
  <mount-target-dns>:/ /mnt/s3files
```

## アクセスポイント

S3 Files では、既存の **S3 アクセスポイント**を利用してアクセス制御を行います。

| 設定項目 | 内容 |
|---|---|
| IAM 連携 | ファイル・オブジェクトレベルの権限管理 |
| 暗号化 | TLS 1.3（転送中）、SSE-S3 または AWS KMS（保存時） |
| 監視・監査 | CloudWatch（モニタリング）、CloudTrail（操作ログ） |

> EFS のアクセスポイント（POSIX ユーザー強制・ルートディレクトリ指定）とは異なり、S3 Files のアクセス制御は **S3 の IAM ポリシー・バケットポリシー**の仕組みをそのまま利用します。

## 料金（目安）

### データアクセス料金

| 操作 | 料金 |
|---|---|
| **小さなファイルの読み取り**（128KB 未満） | ~$0.03 / GB |
| **ストレージへの書き込み** | ~$0.06 / GB |
| **書き込み同期**（高性能ストレージ → S3） | ~$0.03 / GB |
| **大きなファイルの読み取り**（128KB 以上） | 標準 S3 GET リクエスト料金（S3 Files 料金は不要） |

### ストレージ料金

- 高性能ストレージ（EFS ベース）に格納されているアクティブデータに対してストレージ料金が発生
- S3 バケット側は通常の S3 ストレージ料金が適用

### 計算例

100GB のバケットを S3 Files でマウントした場合の月額目安:

| 項目 | 金額 |
|---|---|
| ストレージ料金 | ~$0.255 |
| データアクセス料金 | ~$0.1215 |
| **合計** | **~$0.3765** |

> 詳細・最新価格は [Amazon S3 料金ページ](https://aws.amazon.com/jp/s3/pricing/) を参照

## EFS との比較

| | S3 Files | Amazon EFS |
|---|---|---|
| 基盤ストレージ | Amazon S3（オブジェクトストレージ） | EFS（専用ファイルシステム） |
| プロトコル | NFSv4.1 | NFSv4.1 / NFSv4.0 |
| データの永続先 | S3 バケット | EFS 内（S3 とは別） |
| 既存 S3 データへのアクセス | 直接マウント可能 | 不可（別途データ転送が必要） |
| コスト | S3 料金体系（比較的安価） | EFS 料金体系 |
| ユースケース | S3 上のデータをファイルシステムとして扱いたい | 純粋な共有 NFS ファイルシステムが必要 |

## 全体構成イメージ

```
VPC
├── AZ-a サブネット ─── マウントターゲット ─┐
├── AZ-b サブネット ─── マウントターゲット ─┼── S3 ファイルシステム ── S3 バケット
└── AZ-c サブネット ─── マウントターゲット ─┘        （自動同期）
         ↑
   EC2 / ECS / EKS / Lambda が NFS マウント
```
