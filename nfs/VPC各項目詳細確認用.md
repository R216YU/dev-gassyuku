# AWS VPC 解説

## VPC (Virtual Private Cloud) そのものについて

AWS上に作成する**論理的に隔離された仮想ネットワーク**。

- CIDRブロック（例: `10.0.0.0/16`）でIPアドレス範囲を定義
- リージョン単位で作成される
- 複数のAZをまたがることができる
- デフォルトVPCが各リージョンに自動作成されている

---

## サブネット

VPCのIPアドレス範囲を**さらに分割した区画**。

| 種類 | 概要 |
|------|------|
| パブリックサブネット | インターネットゲートウェイへのルートがある |
| プライベートサブネット | インターネットゲートウェイへの直接ルートがない |

- AZ単位で作成される（1サブネット = 1AZ）
- CIDRはVPCのサブセット（例: `10.0.1.0/24`）
- AWSが各サブネットで5つのIPを予約済み

---

## ルートテーブル

**パケットの転送先を定義するルール集**。

- 各サブネットはいずれか1つのルートテーブルに関連付け
- `0.0.0.0/0 → igw-xxxxx` のようなルートでインターネット接続を制御
- メインルートテーブル（デフォルト）とカスタムルートテーブルがある

```
送信先           ターゲット
10.0.0.0/16     local           ← VPC内通信（自動）
0.0.0.0/0       igw-xxxxxxxx   ← インターネット向け
```

---

## インターネットゲートウェイ (IGW)

**VPCとインターネットをつなぐゲートウェイ**。

- VPCに1つアタッチ（1:1）
- 水平スケーリング・冗長化済みで可用性は高い
- パブリックサブネットのインスタンスはパブリックIP/EIPが必要
- 双方向通信が可能（インバウンド・アウトバウンド両方）

---

## NAT ゲートウェイ

**プライベートサブネットからインターネットへの片方向通信を可能にする**。

- アウトバウンドのみ（インターネット側からの接続は不可）
- パブリックサブネットに配置し、EIPを割り当てる
- AZ単位で作成（高可用性のため各AZに配置推奨）
- マネージドサービスのため運用不要

```
プライベートEC2 → NAT GW（パブリックサブネット）→ IGW → インターネット
```

---

## ネットワーク ACL (NACL)

**サブネット単位のステートレスなファイアウォール**。

| 項目 | 内容 |
|------|------|
| 適用範囲 | サブネット単位 |
| ステート | **ステートレス**（戻りトラフィックも明示的に許可が必要） |
| ルール評価 | ルール番号の小さい順に評価、最初にマッチで決定 |
| デフォルト | 全許可（デフォルトNACL）または全拒否（カスタム） |

---

## セキュリティグループ (SG)

**インスタンス単位のステートフルなファイアウォール**。

| 項目 | 内容 |
|------|------|
| 適用範囲 | ENI（インスタンス）単位 |
| ステート | **ステートフル**（アウトバウンドを許可すれば戻りは自動許可） |
| デフォルト | インバウンド全拒否、アウトバウンド全許可 |
| 参照 | 別のSGをソース/宛先として指定可能 |

**NACLとSGの比較まとめ:**

| | NACL | セキュリティグループ |
|---|---|---|
| レベル | サブネット | インスタンス |
| ステート | ステートレス | ステートフル |
| Allow/Deny | 両方可 | Allowのみ |

---

## エンドポイント (VPC Endpoint)

**インターネットを経由せずにAWSサービスへアクセスする仕組み**。

### Gateway型
- S3、DynamoDBのみ対応
- ルートテーブルにエントリを追加して利用
- 無料

### Interface型 (AWS PrivateLink)
- 多数のAWSサービスに対応（EC2, SSM, SNS等）
- ENIとしてサブネットに作成
- プライベートIPでアクセス
- 有料（時間+データ転送量）

---

## エンドポイントサービス (VPC Endpoint Service)

**自分のサービスを他のVPC/アカウントにPrivateLinkで公開する仕組み**。

```
消費者VPC → Interface Endpoint → エンドポイントサービス → NLB → プロバイダーVPC内のサービス
```

- NLB（またはGateway Load Balancer）をフロントに置いて作成
- 別アカウント・別VPCのユーザーに安全にサービスを提供
- アクセス許可リストで接続を制御可能
- SaaS提供などで活用されるパターン

---

## 全体のアーキテクチャイメージ

```
インターネット
     │
   [IGW]
     │
┌────VPC────────────────────────────┐
│  パブリックサブネット               │
│  [NAT GW] [踏み台EC2]             │
│       │                           │
│  プライベートサブネット             │
│  [アプリEC2] ← SG                 │
│       │                           │
│  [VPC Endpoint] → S3/DynamoDB     │
└───────────────────────────────────┘
     ↑NACL（サブネット境界）
```
